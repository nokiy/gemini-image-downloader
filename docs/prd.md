## 一、产品概述

### 1.1 产品名称
**Gemini Image Downloader** - Gemini 图片批量下载器

### 1.2 产品定位
一款轻量级 Chrome 浏览器扩展，帮助用户一键批量下载 Gemini AI 生成的所有高清图片。

### 1.3 目标用户
- Gemini 网页版用户
- 需要批量保存 AI 生成图片的创作者
- 设计师、内容创作者

---

## 二、用户痛点

| 痛点 | 描述 |
|------|------|
| **操作繁琐** | 需要逐张右键保存图片，效率低下 |
| **图片分散** | 同一对话的图片无法统一管理 |
| **质量不清晰** | 直接保存可能不是最高清版本 |

---

### 2.1 问题定义（第一性原理）

- **根本目标**：用最少步骤把 Gemini 对话中的 AI 图片转成可管理的本地文件，并保持最高可用清晰度
- **本质问题**：扩展无法稳定获取页面图片 URL 时，下载流程被直接阻断
- **关键约束**：Popup 只能通过 Content Script 与页面交互，通信或注入失败会导致「请刷新后重试」

---

## 三、核心需求

### 3.1 MVP 功能（V1.0）

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 一键下载 | P0 | 点击插件图标，自动下载当前对话所有图片 |
| 稳定获取 | P0 | 打开 Popup 后可稳定获取图片列表，失败需给出可执行的恢复路径 |
| 高清优先 | P0 | 获取图片最高分辨率版本 |
| ZIP 打包 | P0 | 将所有图片打包成一个 ZIP 文件 |
| 智能命名 | P1 | 固定命名为 `Gemini_image.zip` |

### 3.2 后续迭代（V2.0+）

| 功能 | 描述 |
|------|------|
| 选择性下载 | 勾选部分图片下载 |
| 下载进度 | 显示下载进度条 |
| 自定义命名 | 用户自定义命名规则 |
| 历史记录 | 下载历史管理 |

## 四、使用场景

### 场景一：创作者素材收集
> 用户在 Gemini 中生成多轮图片后，想要一次性保存所有满意的图片作为素材库。

### 场景二：设计迭代存档
> 设计师使用 Gemini 探索不同风格的设计方案，需要将所有方案图保存以便后续对比。
